# 卡片动画优化与性能指南

## ✨ 已完成的优化

### 一、卡片动画效果优化

#### 1. 悬停效果 (Hover Effect)
- **功能**：鼠标悬停时卡片轻微放大并变色
- **参数配置：
  - `hoverScale = 1.05f` - 悬停放大比例
  - `hoverDuration = 0.15f` - 动画时长
  - `hoverColor` - 悬停时背景色变化

#### 2. 优化翻转动画
- **使用 AnimationCurve**：可在 Inspector 自定义翻转曲线
- **默认曲线**：EaseInOut - 平滑的缓入缓出效果
- **翻转速度**：优化为 0.35 秒，响应更灵敏

#### 3. 发光效果支持
- 预留 `glowEffect` 引用字段
- 可添加发光 Image 增强视觉反馈

### 二、游戏胜利界面 (VictoryManager)

#### 胜利动画流程
```
1. 面板淡入 (0.5秒)
   ↓
2. 彩带粒子特效
   ↓
3. 标题弹动缩放
   ↓
4. 星星评分依次弹出 (每颗0.3秒间隔)
   ↓
5. 统计数据依次显示
   ↓
6. 按钮淡入
```

#### 星星评分系统
根据步数和时间自动计算星级：
| 表现 | 星级 | 条件 |
|------|------|------|
| 完美 | ⭐⭐⭐ | 步数 ≤ 1.5倍完美步数，时间 ≤ 2倍完美时间 |
| 良好 | ⭐⭐ | 步数 ≤ 2倍完美步数，时间 ≤ 3倍完美时间 |
| 一般 | ⭐ | 其他情况 |

#### 胜利界面配置
在 Canvas 下创建胜利面板结构：
```
VictoryPanel (添加 VictoryManager 脚本)
├── TitleText ("恭喜通关！")
├── Stars (3个星星Image)
├── ScoreText
├── MovesText
├── TimeText

```

### 三、性能优化 (PerformanceManager)

#### 1. 帧率控制
- **目标帧率**：60 FPS
- **垂直同步**：可配置启用/禁用
- **最大排队帧**：2 帧，减少输入延迟

#### 2. 对象池管理 (Object Pooling)
- 预分配 24 张卡片对象池
- 避免频繁 Instantiate/Destroy 造成的 GC
- 可在 Inspector 配置池大小

#### 3. 内存优化
- 启动时强制 GC 回收
- 提供手动 GC 接口
- 支持低质量模式切换

#### 4. 性能监控
- 可启用 FPS 显示
- 实时帧率统计
- 配置显示更新间隔

## 🎮 Unity 配置步骤

### 步骤 1：配置卡片动画

1. 选中卡片预制体
2. 在 Card 组件 Inspector 中配置：
   - ✅ `Enable Hover Effect` - 启用悬停效果
   - `Hover Scale` = 1.05
   - `Hover Duration` = 0.15
   - `Flip Curve` - 保持默认或自定义曲线
   - `Flip Duration` = 0.35

### 步骤 2：创建胜利界面

1. 在 Canvas 下创建 Panel，命名为 "VictoryPanel"
2. 添加以下子元素：
   - Text: TitleText ("恭喜通关！")
   - 3个 Image: Star1, Star2, Star3
   - Text: ScoreText, MovesText, TimeText
   - Button: PlayAgainButton, MainMenuButton
3. 添加 `VictoryManager.cs` 脚本到 VictoryPanel
4. 拖拽所有引用到 Inspector

### 步骤 3：配置性能管理

1. 创建 Empty GameObject，命名为 "PerformanceManager"
2. 添加 `PerformanceManager.cs` 脚本
3. 配置参数：
   - `Target Frame Rate` = 60
   - ✅ `VSync Enabled`
   - `Initial Card Pool Size` = 24
   - ☐ `Show FPS` (测试时可勾选)

### 步骤 4：测试验证

1. 运行游戏
2. 悬停在卡片上 - 观察放大和变色效果
3. 点击翻转卡片 - 验证翻转动画流畅
4. 完成所有配对 - 验证胜利界面动画

## 🐛 常见问题排查

### 问题 1：动画卡顿
**可能原因：**
- 帧率被限制
- 后台有其他程序占用资源
- GC 频繁回收

**解决方案：**
- 检查 PerformanceManager 的帧率设置
- 关闭不必要的后台程序
- 启用对象池减少 GC

### 问题 2：悬停效果不生效
**可能原因：**
- `enableHoverEffect` 未勾选
- 卡片被其他 UI 遮挡
- `isMatched` 或 `isFlipped` 为 true

**解决方案：**
- 确认 `Enable Hover Effect` 已勾选
- 检查 Canvas 层级
- 确认卡片状态

### 问题 3：胜利界面不显示
**可能原因：**
- 事件未正确注册
- 面板初始状态错误
- 引用未正确设置

**解决方案：**
- 检查 GameManager 是否存在
- 确认 VictoryPanel 初始为 inactive
- 检查所有 Text/Button 引用

### 问题 4：帧率过低
**可能原因：**
- 开启了过多特效
- 分辨率过高
- 设备性能不足

**解决方案：**
- 减少粒子效果数量
- 降低渲染分辨率
- 切换到低质量模式
