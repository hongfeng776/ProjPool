二、层级设置 (Layer Setup)
===========================

在 Unity 中点击 Layers → Edit Layers，添加以下层：

- Layer 8: Interactive (可交互物品)
- Layer 9: Player      (玩家)


三、物理层设置 (Physics Layer Collision)
=========================================

Edit → Project Settings → Physics

确保以下层之间的碰撞：
- Player 与 Default (可以和环境碰撞)
- Boss 与 Default (可以和环境碰撞)
- Interactive 可以与 Raycast 交互 (不需要物理碰撞)

可以关闭不需要的层碰撞以提高性能。


四、动画控制器建议 (Animator)
===============================

如果使用角色动画，建议设置以下参数：

Player Animator 参数：
- int State (0=Working, 1=Fishing, 2=GettingCaught)
- trigger Work
- trigger Fish
- trigger Caught

Boss Animator 参数：
- float Speed
- trigger Angry


五、游戏平衡数值参考
=====================

摸鱼物品配置参考：
| 物品   | 持续时间 | 基础分 | 风险值 | 说明 |
|--------|----------|--------|--------|------|
| 手机   | 2.5秒    | 15分   | 0.7    | 中高风险 |
| 游戏机 | 4.0秒    | 30分   | 0.9    | 高风险高回报 |
| 漫画   | 3.0秒    | 20分   | 0.5    | 中等 |
| 零食   | 2.0秒    | 10分   | 0.3    | 低风险 |
| 茶水   | 1.5秒    | 5分    | 0.2    | 最安全 |

游戏设置：
- 初始血量: 3
- 被抓伤害: 1
- 被抓扣分: 10% 当前分
- 连击加成: 每连击 +10% 分
- 最大连击: 10
- 游戏时长: 120秒
- 老板检测时间: 0.5秒

6.5 小游戏完成后的分数计算
----------------------------

PhoneGame: 基础分 + 连击加成 + 风险系数
- 成功点击加分，避开"工作邮件"扣分
- 连击: 每连击 +2 分，最多 +20 分

CardGame: 配对分 + 效率加成 + 时间加成
- 每个配对: +25 分
- 效率加成: (最少步数 - 实际步数) × 5
- 时间加成: 剩余时间 × 2

SnackGame: 零食分 + 饱腹加成 + 完成加成
- 每个零食基础分
- 饱腹加成: 饱腹度 × 20
- 完成加成: 完成度 × 30


七、排行榜配置
================

7.1 排行榜管理器
------------------

在 GameManager 对象下添加 [LeaderboardManager] 子对象：
├── [LeaderboardManager] (空物体)
│   └── LeaderboardManager.cs (脚本)

排行榜支持以下功能：
- 保存游戏分数到 SQLite 数据库
- 查询最高分 (GetTopScores)
- 查询玩家历史分数 (GetPlayerScores)
- 查询玩家排名 (GetPlayerRank)
- 清空排行榜 (ClearLeaderboard)

7.2 排行榜数据存储
--------------------

数据库文件位置：
- Editor: Assets/OfficeFishing.db
- 平台: Application.persistentDataPath/OfficeFishing.db

排行榜表结构：
- Id: 唯一标识
- PlayerId: 玩家 ID
- PlayerName: 玩家名称
- Score: 分数
- GameMode: 游戏模式 (Default/Phone/Card/Snack)
- PlayedAt: 游玩时间

7.4 分数保存流程
--------------------

游戏结束时自动保存：
1. OfficeFishingGameManager.EndGame() 调用 SaveScoreToDatabase()
2. LeaderboardRepository.AddScore() 插入数据库
3. LeaderboardController.RefreshLeaderboard() 刷新显示

8.4 道具触发方式
------------------

道具可以通过以下方式获取：

- 关卡进度奖励

使用道具：
- 点击道具槽位使用
- 或按快捷键 1-4 使用对应位置的道具

9.4 关卡进度保存
------------------

关卡进度通过 PlayerPrefs 保存：
- Key: "UnlockedLevel"
- Value: 已解锁的最高关卡索引 (0 表示第 1 关)

通关条件：
- 达到目标分数
- 或在限定时间内存活


十、游戏结束界面增强
==================

10.1 游戏结束面板配置
-----------------------

更新 GameOverPanel 添加以下元素：

├── [GameOverPanel] (Image - 面板)
│   ├── BG (Image - 半透明背景)

-----------------------------

在 GameHUDController 中设置：

- leaderboardButton: 关联到排行榜按钮


十一、测试步骤
=============

1. 点击 Play 运行场景
2. 鼠标悬停在物品上，应该看到高亮效果
3. 点击非电脑物品开始摸鱼，弹出小游戏面板
4. 完成小游戏后获得分数，底部显示进度条
5. 点击电脑停止摸鱼，进度条消失
6. 摸鱼时老板进入检测范围，顶部显示警告
7. 被抓后扣血扣连击，2秒后恢复
8. 使用道具：点击道具槽位或按 1-4 键
9. 验证道具效果：老板变慢、玩家隐身等
10. 血量归零或时间到，显示游戏结束面板
11. 查看游戏结束界面：显示本次摸鱼值和历史最高
12. 暂停按钮可以暂停/继续游戏
13. 查看主菜单排行榜，确认分数已保存
14. 切换关卡，验证难度变化
